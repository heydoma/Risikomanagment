\section{Annahmen: Phasen/Artefakte des DASC-PM}

\notiz{
    Wie einleitend genannt, soll sich diese Arbeit mit der Strategieentwicklung für das Riskmanagement in Data-Science Projekten 
    entlang des DASC-PM als Grundlage befassen.
    Dafür soll das Werkzeug der, in der Softwareentwicklung üblichen, Quality-Gates als Ergänzungsvorschlag für das Prozessmodell 
    DASC eingeführt werden.
}

\notiz{
    Das DASC-PM ist aktuell das modernste agile Modell, welches auch umfangreich und offen zugänglich definiert ist.
    Damit bietet es ein gutes Fundament für die weitere Integration von Risikomanagement Schritten bzw. Prozessen.
}

\notiz{ STRUKTUR: 
        - Warum entlang des DASC-PM? -> Modern, Agil, Definiert
        - Eigenschaften von Quality-Gates in der Softwareentwicklung
        - Erwartungen an Quality-Gates im Riskmanagement 
        - Die sich daraus ergebende Herleitung warum Quality-Gates eine sinnvolle Ergänzung sind für das DASC-PM
        - Übergang zur eigentlichen Integration
        }
